# GitHub Desktop Workflow 

1. Open GitHub Desktop.
2. Pull the latest changes before editing.
3. Create a new branch with a short descriptive name.
4. Edit or add the script file.
5. Test the script locally using safe sample data.
6. Review changed files in GitHub Desktop.
7. Commit with a clear message.
8. Push the branch to GitHub.
9. Open a pull request.
10. Request review from a department reviewer or IT.
11. Merge only after approval.
12. Pull the merged change locally.

## Good commit examples

- Add layer cleanup option to drawing startup script
- Fix incorrect beam weight calculation
- Add error handling when spreadsheet is unavailable

## Poor commit examples

- Update
- Changes
- Fixed stuff
- New version
